###git设置代理
我在公司使用git想将本地代码push到远程的时候，一直报

connect to host github.com port 22: Connection timed out
fatal: Could not read from remote repository.

这个错误，原因是我在公司是通过http代理上网的，而push到远程的时候git应该是不通过http代理的(这是我猜的，如果有误欢迎指出)，所以我们需要先在git中设置代理：

但这样设置代理后push到远程貌似只能通过http方式，不支持SSH方式push（我电脑上经过测试是这样的，如果哪位大神可以找到方法，麻烦告知一下）。要修改push的方式，进入.git文件夹，打开config文件，找到url那一项，那url的值改成

https://github.com/yourname/yourProj.git

保存后重新进行push就行了，但这种方式每次push的时候都要输入用户名和密码，比较麻烦。而通过SSH方式push不用输用户名和密码（前提是你先配置好SSH），如何配置SSH自行百度，也很简单，但在公司又没法通过SSH push，所以很蛋疼，没找到两全其美的办法。手动哭脸。
